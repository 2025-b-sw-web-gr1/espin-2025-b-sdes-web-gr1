### Descripción de carpetas

* **assets/**
  Recursos estáticos utilizados en el proyecto, como imágenes, íconos o archivos de apoyo.

* **Class 002 – Class 008**
  Carpetas correspondientes a las clases del curso. En estas sesiones se trabajó principalmente en:

  * Comprender cómo funcionan los **estilos CSS**
  * Estructura y semántica de páginas web con **HTML**
  * Uso de **JavaScript** para agregar interactividad
  * Integración de HTML, CSS y JS en proyectos simples

* **Ejemplo/**
  Contiene ejemplos prácticos desarrollados durante el curso para reforzar conceptos vistos en clase.

* **Examen 001/**
  Evaluación práctica orientada a la **documentación de APIs REST**. Incluye:

  * Definición de la API mediante **OpenAPI 3.0** en `docs/openapi.yaml`
  * Uso de **Swagger UI** para generar una interfaz interactiva (`swagger-ui/examen.html`)
  * Pruebas de endpoints directamente desde la interfaz Swagger
  * Un `README.md` específico con la explicación del examen y su objetivo académico

  Este examen evalúa la comprensión de estándares de documentación, estructura de una API y buenas prácticas técnicas.

* **Proyect 001/**
  Proyecto práctico enfocado en el diseño y organización de una **API de dominio específico (fútbol)**. Contiene:

  * Datos estructurados en formato **JSON** (`bruno.json`)
  * Organización de recursos por entidades (`players`, `teams`)
  * Carpeta `swagger/` con el archivo `football-api.yaml`, donde se documenta la API usando **Swagger/OpenAPI**

  El proyecto integra conceptos de modelado de datos, organización de recursos y documentación técnica de servicios.

* **index.html**
  Página principal del repositorio, utilizada como punto de entrada o demostración de los contenidos desarrollados.
